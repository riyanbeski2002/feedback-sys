In `src/features/hotels/components/hotel-card.tsx`, find the two lines using the hardcoded `rgba(59, 130, 246, 0.5)` blue in the Framer Motion flash animation (approximately lines 61-62).

Replace both occurrences:

BEFORE:
```tsx
boxShadow: isFlashing ? "0 0 20px rgba(59, 130, 246, 0.5)" : "none",
borderColor: isFlashing ? "rgba(59, 130, 246, 0.5)" : "inherit"
```

AFTER:
```tsx
boxShadow: isFlashing ? "0 0 20px hsl(171 53% 64% / 0.5)" : "none",
borderColor: isFlashing ? "hsl(171 53% 64% / 0.5)" : "inherit"
```

Why `hsl(171 53% 64% / 0.5)` not `var(--primary)`: Framer Motion `animate` prop receives computed values and does not resolve CSS variables at animation time. The HSL value `hsl(171 53% 64%)` is the resolved value of `var(--primary)` as defined in `globals.css` under `@theme inline`. The `/ 0.5` alpha applies the same 50% opacity as the original rgba.

Do NOT change any other lines. This is a two-line surgical edit.
  </action>
