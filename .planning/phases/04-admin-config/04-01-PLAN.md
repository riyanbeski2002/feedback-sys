# Plan: 04-01 - Admin & Config

## Phase Boundary
Providing tools for management and system-wide configuration.

## Goals
1. **Admin Dashboard**: Create a central operational view with summary metrics and flagged hotel lists.
2. **Settings Page**: Implement a configuration panel to manage scoring weights, thresholds, and communication rules.
3. **Data Integrity**: Ensure configuration changes are persisted and reflected (optionally) in scoring calculations.

## Tasks

### 1. Admin Dashboard (`/admin`)
- [ ] Create `src/app/(dashboard)/admin/page.tsx`.
- [ ] Implement `MetricCards` for summary stats (Total Feedbacks, Avg Score, Flagged Count).
- [ ] Build `FlaggedHotelsTable` component.
- [ ] Add `RecentFeedbackFeed` component.

### 2. Settings & Config (`/settings`)
- [ ] Create `src/app/(dashboard)/settings/page.tsx`.
- [ ] Create `ConfigForm` component in `src/features/admin/components/config-form.tsx`.
- [ ] Implement `updateConfig` server action in `src/features/admin/actions/update-config.ts`.
- [ ] Add validation logic for weights (must sum to 1.0).

### 3. Data Integration
- [ ] (Optional/Bonus) Refactor `calculateWeightedScore` in `src/features/feedback/lib/scoring.ts` to use weights from `feedback_config` if available.

## Validation Strategy
- **Dashboard**: Verify that hotels with score < 2.0 appear in the "Flagged Hotels" list.
- **Config**: Update scoring weights in `/settings` and verify the values are saved in the `feedback_config` table.
- **Rules**: Toggle a communication channel and verify the setting is persisted.
