<task type="auto">
  <name>Task 2: Migrate globals.css to correct Tailwind v4 token structure</name>
  <files>src/app/globals.css</files>
  <action>
    Rewrite `src/app/globals.css` to use the correct Tailwind v4 + shadcn/ui pattern. Three structural changes are required:

    1. **Move `:root` and `.dark` blocks outside `@layer base`** — place them at the top of the file, directly after `@import "tailwindcss"`. In v4, `@layer base` has lower cascade priority than utility classes; CSS custom properties declared inside it can be overridden unexpectedly. They must live outside any layer.

    2. **Add `hsl()` wrappers to all bare HSL values** — every value like `180 10% 98%` must become `hsl(180 10% 98%)`. Tailwind v4 uses values literally; bare HSL numbers are not valid CSS color values and will not resolve.

    3. **Change `@theme {}` to `@theme inline {}`** — without `inline`, the engine attempts OKLCH color space conversion on token values. When the value is `var(--primary)` (a CSS variable reference, not a literal color), the conversion receives an unresolvable value and silently emits an empty string. `@theme inline` preserves the `var()` reference as-is.

    **The second `@layer base` block** (the one containing `* { @apply border-border }` and `body { @apply bg-background text-foreground antialiased }`) stays inside `@layer base` exactly as it is. Do not move or modify it.

    **Do NOT change any HSL color values themselves.** The scope of this task is structural migration only — moving blocks, adding `hsl()` wrappers, and changing `@theme` to `@theme inline`. Visual regressions are acceptable in Phase 6; Phase 8 will address all color values.

    The final structure must be:
    ```
    @import "tailwindcss";

    :root {
      --background: hsl(180 10% 98%);
      /* ... all tokens with hsl() wrappers ... */
    }

    .dark {
      --background: hsl(180 5% 5%);
      /* ... all dark tokens with hsl() wrappers ... */
    }

    @theme inline {
      --color-border: var(--border);
      /* ... all @theme mappings unchanged ... */
    }

    @layer base {
      * { @apply border-border; }
      body { @apply bg-background text-foreground antialiased; }
    }
    ```

    Read the current `src/app/globals.css` in full before writing. The current file has all `:root` and `.dark` tokens (lines 37–99), the `@theme {}` block (lines 3–34), and the utility `@layer base` block (lines 102–109). Preserve every token name and every color value — only change structure.
  </action>
