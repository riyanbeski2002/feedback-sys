<decisions>
## Implementation Decisions

### Verification Scope
- Verify **all 13 Tech Stories** (including TS-13 Sentiment Analysis, which is optional/stretch)
- Primary reference for truth: **PRD first, MVP second** — stories should match PRD intent; deviations in MVP from both are flagged separately
- **Flag uncovered PRD requirements** — identify any PRD requirement that no Tech Story addresses
- **Validate inter-story dependency chains** — catch stories that reference each other incorrectly or have circular logic

### Output Format
- Primary deliverable: **VERIFICATION-REPORT.md** — structured gap report document saved in the phase directory
- **Summary table at top** showing per-story status: `Aligned` / `Minor gaps` / `Major gaps` / `Planned, not built`
- Detailed per-story findings below the summary table
- Each gap entry includes a **recommended action**
- **Internal consistency check** within each story (catch logic errors like the TS-1/TS-2 timing issue already fixed)

### Gap Severity — 3 Tiers
- **Critical** — PRD requirement has no corresponding Tech Story, or a story has a fundamental logic error that would break implementation
- **Minor** — Story describes something differently from MVP but PRD requirement is still satisfied; or wording deviates from what was built
- **Cosmetic** — Naming, style, or formatting inconsistencies only; no logical impact

### Gap Classification Rules
- Story describes behavior implemented differently in MVP (PRD still satisfied) → **Minor gap**, note the deviation and suggest aligning
- Feature described in a story but NOT yet built in MVP → **"Planned, not built"** — not a gap (intentional for future phases)
- Effort estimates that look significantly off vs. actual MVP complexity → flag with note
- PRD requirement with no Tech Story → **Critical gap**

### Action on Gaps
- **Critical gaps** → Write a new Tech Story (TS-14+) and add it directly to TECH-STORIES.md marked `[ADDED — GAP CLOSURE]`
- **Minor gaps** → Document in report only; no file changes
- **Cosmetic gaps** → Auto-fix in TECH-STORIES.md (wording clarifications, typo corrections)
- The executor produces the report AND applies auto-safe corrections (cosmetic fixes + new stories for critical gaps); Critical gaps are also clearly called out for manual review

### Claude's Discretion
- The exact structure of VERIFICATION-REPORT.md sections (as long as it has the summary table + per-story breakdown)
- How to order findings within each severity tier
- Whether to include a "What was verified against" preamble listing the source documents used

</decisions>
