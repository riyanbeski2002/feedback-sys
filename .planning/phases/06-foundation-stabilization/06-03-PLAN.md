    **Step 1 — Add dotenv to devDependencies:**
    Run:
    ```
    npm install --save-dev dotenv
    ```
    This adds `dotenv` to `devDependencies` in `package.json`. Do not change any other dependencies.

    **Step 2 — Create `src/scripts/tsconfig.json`:**
    Create the file with this exact content to override the root tsconfig's `"module": "esnext"` setting for the seed script. Without this override, `ts-node` picks up the root config and fails because the seed uses CommonJS `require()` syntax.

    ```json
    {
      "extends": "../../tsconfig.json",
      "compilerOptions": {
        "module": "commonjs",
        "moduleResolution": "node"
      }
    }
    ```

    **Step 3 — Update the seed npm script in `package.json`:**
    The current seed script is `"seed": "ts-node src/scripts/seed.ts"`. Update it to use the local tsconfig:
    ```
    "seed": "ts-node --project src/scripts/tsconfig.json src/scripts/seed.ts"
    ```
    This ensures ts-node uses the CommonJS override, not the root ESM config.
  </action>
